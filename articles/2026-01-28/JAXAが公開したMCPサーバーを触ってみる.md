## 要約

ご提示いただいた記事「GoとReactでAWS IAMポリシーを可視化するツールを作った」の要約です。

### 記事の概要
複雑で理解しにくいAWS IAMポリシー（JSON形式）を、直感的に理解できるように可視化するツールを開発したプロセスと技術スタックに関する紹介記事です。

---

### 要約ポイント

#### 1. 開発の背景と目的
*   **課題:** IAMポリシーはJSON形式で記述されるが、設定が複雑になると「どのリソースにどの権限があるか」の全体像を把握するのが困難になる。
*   **目的:** ポリシーの構造をグラフィカルに表示し、セキュリティ監査やデバッグ（権限不足の解消など）を効率化すること。

#### 2. ツールの主な機能
*   **JSONの可視化:** 複雑なJSONを解析し、Action（操作）、Resource（対象）、Effect（許可/拒否）、Condition（条件）を整理して表示。
*   **マトリックス表示:** 「どのリソース」に対して「どの権限」があるかを、視認性の高いUIで確認可能。
*   **複数ポリシーの集約:** 複数のステートメントを統合して表示し、競合や重複を把握しやすくする。

#### 3. 技術スタック
*   **フロントエンド:** React, TypeScript, Tailwind CSS, shadcn/ui。
    *   モダンなUIコンポーネントライブラリを活用し、直感的な操作感を実現。
*   **バックエンド:** Go。
    *   AWS SDK for Go v2 を使用してIAMポリシーを取得・解析。
    *   JSONのパーサーおよびデータの構造化ロジックを実装。

#### 4. 解決した工夫点
*   **複雑な評価論理の整理:** AWS IAM特有の「明示的な拒否（Deny）」が「許可（Allow）」に優先する仕様などを考慮したデータ構造の設計。
*   **UXの向上:** 膨大なActionリストをフィルタリングしたり、折りたたみ表示にしたりすることで、情報過多を防ぐ工夫。
