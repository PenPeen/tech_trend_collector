### 記事の概要
GitHub Actionsを利用して、GitHubのリポジトリにコードをプッシュした際、自動的にAWSのS3とCloudFrontへ静的コンテンツをデプロイ（更新）するCI/CD環境の構築手順を解説した初心者向けのガイドです。

---

### 主な内容と手順

#### 1. AWS側の構築（S3・CloudFront）
*   **S3:** 静的ウェブサイトのファイルを格納するバケットを作成。
*   **CloudFront:** S3をオリジンとして配信設定を行い、高速化とセキュリティを向上。
*   **OAC（Origin Access Control）:** S3バケットへのアクセスをCloudFront経由のみに制限する安全な設定を推奨。

#### 2. IAMユーザーの作成
*   GitHub ActionsからAWSを操作するための専用ユーザーを作成。
*   `AmazonS3FullAccess` および `CloudFrontFullAccess` の権限を付与し、「アクセスキー」と「シークレットアクセスキー」を取得。

#### 3. GitHub Secretsの設定
*   GitHubのリポジトリ設定（Secrets and variables）に、AWSの認証情報やバケット情報を環境変数として登録。
    *   `AWS_ACCESS_KEY_ID`
    *   `AWS_SECRET_ACCESS_KEY`
    *   `AWS_S3_BUCKET_NAME`
    *   `AWS_CLOUDFRONT_DISTRIBUTION_ID`

#### 4. GitHub Actionsの設定ファイル作成
*   リポジトリ内に `.github/workflows/deploy.yml` を作成し、以下の処理を定義。
    1.  **Checkout:** リポジトリからコードを取得。
    2.  **Configure AWS Credentials:** AWSへのログイン認証。
