## 要約

ご提示いただいたQiitaの記事「AWS CLI 実行時に IAM ユーザーを切り替える手順（名前付きプロファイル）」の要約です。

### 記事の概要
AWS CLIを利用する際、複数のAWSアカウントや異なる権限を持つIAMユーザーを、**「名前付きプロファイル（Named Profiles）」**という機能を使って効率的に切り替えて管理する方法について解説しています。

---

### 主な内容

#### 1. 名前付きプロファイルとは
AWS CLIの設定（アクセスキーやリージョンなど）に名前を付けて保存できる機能です。これにより、開発環境・本番環境といった複数の環境をコマンド一つで使い分けることが可能になります。

#### 2. プロファイルの設定方法
以下のコマンドを実行し、対話形式で情報を入力します。
```bash
aws configure --profile プロファイル名
```
*   **設定項目:** Access Key ID, Secret Access Key, Region, Output format
*   **確認方法:** `~/.aws/credentials` や `~/.aws/config` ファイルに追記されます。

#### 3. コマンドでの利用方法
各コマンドの末尾に `--profile` オプションを付けることで、指定したプロファイルの権限で実行できます。
```bash
aws s3 ls --profile プロファイル名
```

#### 4. 環境変数による切り替え
コマンドごとにオプションを付けるのが手間な場合は、環境変数を設定することで、そのターミナルセッション中のデフォルトプロファイルを固定できます。
```bash
export AWS_PROFILE=プロファイル名
```

### まとめ・メリット
*   **誤操作の防止:** 複数のアカウントを扱う際、意図しない環境（本番環境など）に対して操作を行うリスクを減らせます。
*   **利便性の向上:** 設定ファイルを書き換えることなく、スムーズに権限の切り替えが可能になります。
